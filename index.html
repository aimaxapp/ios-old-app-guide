A Update Locker（阻止商店更新）最推薦

LSPosed 模組，阻止 Google Play 等應用商店更新你選定的 App。

  1. 下載並安裝 Update Locker APK
  2. 打開 LSPosed → 啟用 Update Locker 模組
  3. 在 LSPosed 中設定作用域 → 勾選 Google Play 商店
  4. 打開 Update Locker → 選擇你要鎖定的 App（如 Uber Eats）
  5. 重啟手機或多次強制停止 Google Play
  6. 完成！選定的 App 不會再被 Play Store 更新

B 反編譯 APK 移除更新檢查（最徹底）

直接修改 Uber Eats 的 APK，刪除強制更新的代碼。這是最徹底的方法，連伺服器端的版本檢查回應都可以忽略。

  1. 在電腦上安裝 APKTool 和 jadx
  2. 從手機提取 Uber Eats APK（或從 APKMirror 下載）
  3. 用 jadx 反編譯，搜尋以下關鍵字：
    ForceUpdate VersionCheck MinimumVersion UnsupportedOS no longer supported
  4. 用 APKTool 反編譯 smali 代碼：
    java -jar apktool.jar d uber_eats.apk -o uber_eats_src
  5. 找到更新檢查的 smali 檔案，將版本比較邏輯改為永遠返回 true（通過）
  6. 重新打包：
    java -jar apktool.jar b uber_eats_src -o uber_eats_mod.apk
  7. 用 uber-apk-signer 簽名：
    java -jar uber-apk-signer.jar -a uber_eats_mod.apk
  8. 安裝修改後的 APK 到手機
下載 APKTool（GitHub） APK 反編譯/重新打包工具 下載 jadx（GitHub） Java 反編譯器 · 搜尋更新檢查代碼 下載 uber-apk-signer（GitHub） APK 重新簽名工具
💡 小技巧：用 jadx 打開 APK 後，按 Ctrl+Shift+F 全域搜尋「force update」或「minimum version」，通常可以快速定位到更新檢查的代碼位置。

C Xposed 模組攔截更新彈窗（自動關閉）進階

自製 LSPosed 模組，自動偵測並關閉所有包含「更新」「不支援」等關鍵字的彈窗，同時偽裝 Android 版本號。

💡 這個模組的功能：
・偽裝 Android 版本為 Android 15 (SDK 35)
・自動關閉包含「更新」「update」「不支援」「not supported」等關鍵字的 AlertDialog
・攔截所有 Dialog.show()，檢查內容後自動 dismiss
・攔截 Google Play In-App Update API
・攔截名稱包含 update/upgrade 的 Activity
・攔截 WebView 載入更新相關 URL
・Hook Build.VERSION 和 System.getProperty

使用方法：

  1. 下載下方的 Xposed 模組源碼
  2. 用 Android Studio 編譯成 APK
  3. 安裝到手機
  4. 打開 LSPosed → 啟用模組
  5. 設定作用域 → 勾選 Uber Eats（com.ubercab.eats）
  6. 重啟手機或強制停止 Uber Eats
  7. 重新打開 Uber Eats 測試

模組核心代碼（Java）：

// UberEatsUpdateBlocker.java
// 偽裝 Android 版本
XposedHelpers.setStaticIntField(
  Build.VERSION.class, "SDK_INT", 35); // Android 15
XposedHelpers.setStaticObjectField(
  Build.VERSION.class, "RELEASE", "15");

// 攔截 AlertDialog
XposedHelpers.findAndHookMethod(
  AlertDialog.Builder.class, "show",
  new XC_MethodHook() {
    protected void beforeHookedMethod(MethodHookParam p) {
      // 檢查 message 是否包含更新關鍵字
      // 如果是，param.setResult(null) 阻止顯示
    }
  });

// 攔截 Dialog.show() 自動關閉
XposedHelpers.findAndHookMethod(
  Dialog.class, "show",
  new XC_MethodHook() {
    protected void afterHookedMethod(MethodHookParam p) {
      // 遞迴搜尋 Dialog 中所有 TextView
      // 如果包含「更新」「not supported」等關鍵字
      // 自動 dialog.dismiss()
    }
  });
完整源碼（需自行編譯） Java · 需要 Android Studio + Xposed API
⚠️ 編譯環境需求：
・Android Studio（任意版本）
・Xposed API 依賴：compileOnly 'de.robv.android.xposed:api:82'
・如果你不會編譯，可以直接用方案 A（Update Locker）+ 搭配 Android 版本偽裝

🎯 推薦組合策略

針對 Uber Eats 這種同時有「商店更新 + App 內部強制更新」的 App，建議組合使用：

  1. Update Locker（方案 A）→ 阻止 Google Play 自動更新
  2. 反編譯 APK（方案 B）→ 移除 App 內部的版本檢查代碼
  3. 或用 Xposed 模組（方案 C）→ 自動攔截更新彈窗 + 偽裝系統版本
💡 Android 的優勢：已 Root 的 Android 可以做到幾乎任何修改，不像 iOS 受到硬體和伺服器端的雙重限制。在 Android 上成功繞過強制更新的機率遠高於 iOS。
